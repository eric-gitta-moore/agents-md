1. 使用简体中文对话
2. 不要在修改完代码之后马上跑全量的 eslint 或者 vue-tsc 类似的检查，全量检查很浪费时间
3. node 环境使用 fnm 管理; golang 环境使用 g 管理; python 环境使用 uv 管理
4. 对于我自己来说，我的终端使用的命令行环境是 zsh
5. 仔细检查注释，比较关键的函数/逻辑都必须加上易懂的注释
6. 仔细检查变更，在经历多轮变更之后不要留下补丁叠补丁式的代码或注释，如有，则立刻统一逻辑
7. 飞书文档的相关搜索/写入使用 larksuite-tools(skill) ，但是lark/飞书文档的读取一定要使用 mcporter cli
8. 在使用浏览器操作/调试的时候，可以使用 agent-browser(skill) 但是要记得加上 --cdp 9222 参数
9. 注意工程化思维，包括但不限于组件/模块/hooks的抽象和封装思维，也要注意实现的可读性和可维护性
10. 在碰到需要手搓某个模块/功能的时候，可以尝试看看仓库类是否有能够复用的实现，或者网络上有没有第三方包/依赖可复用
11. 在使用各种工具(包括但不限于 cli/skill/mcp )的时候，如果有不清楚的地方一定记得获取帮助(比如 cli 的话就及时使用 --help )
12. 我的各个项目工作目录都是 ~/workspace/<项目名>，如有需要 clone 项目请放在这个目录下面
